## 🛠️ **구축된 자동화 시스템**

### 1️⃣ **자연어 UX 요청 파서** (`parse_ux_request.sh`)

**기능**:
- 자연어 입력을 분석하여 자동으로 JIRA 이슈 생성
- 스마트 카테고리 감지 및 분류
- 우선순위 자동 설정
- 적절한 라벨 자동 태깅

**감지 키워드**:
```yaml
불만: "문제야", "이상해", "버그", "안돼", "작동안해", "짜증", "불편"
개선: "~하면 좋겠어", "바꿔줘", "개선", "수정해줘"
UX: "사용하기 어려워", "터치하기 어려워", "보기 힘들어"
디자인: "폰트", "색상", "크기", "간격", "레이아웃", "애니메이션"
```

**실행 결과**:
```
🤖 자연어 UX 요청 감지됨!
📝 제목: [자동 추출된 제목]
🏷️ 카테고리: [감지된 카테고리]
⚡ 우선순위: [자동 설정된 우선순위]

✅ JIRA 이슈 KAN-XX 생성 완료!
🔗 https://beyond-app.atlassian.net/browse/KAN-XX
```

### 2️⃣ **Git-JIRA 통합 워크플로우** (`git_jira_commit.sh`)

**기능**:
- Git 커밋과 JIRA 업데이트를 한 번에 처리
- 자동 GitHub 링크 연결
- JIRA 이슈 상태 자동 변경 (Done, In Progress, Comment)
- 타임스탬프 자동 기록

**사용법**:
```bash
# 작업 완료 시 자동 Done 처리
./scripts/git_jira_commit.sh "작업 완료 설명" "KAN-XX" "done"

# 진행 중 업데이트
./scripts/git_jira_commit.sh "작업 진행 상황" "KAN-XX" "in-progress"

# 단순 커밋 정보 추가
./scripts/git_jira_commit.sh "코드 수정 내용" "KAN-XX"
```

### 3️⃣ **Claude Code 영구 자동화 지침**

**CLAUDE.md 필수 워크플로우**:
- 모든 불만/개선 요청 감지 시 자동 JIRA 등록
- 문제 해결 후 자동 JIRA 완료 처리
- 완전 자동화된 이슈 관리 사이클
