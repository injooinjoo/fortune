# 🎨 UX/디자인 개선 요청 가이드

## 📋 자동 JIRA 생성 시스템

이제 UX나 디자인 개선 요청을 **자동으로 JIRA에 추가**할 수 있습니다!

### 🚀 사용 방법

#### 1. 기본 사용법
```bash
./scripts/create_ux_request.sh "제목" "상세내용" "카테고리"
```

#### 2. 카테고리별 예시

**🔤 폰트/타이포그래피**
```bash
./scripts/create_ux_request.sh "메인 제목 폰트 크기 조정" "메인 화면 제목이 너무 작아서 가독성이 떨어져요" "font"
```

**🎨 색상/테마**
```bash
./scripts/create_ux_request.sh "다크모드 색상 대비 개선" "다크모드에서 일부 텍스트가 잘 안 보여요" "color"
```

**✨ 애니메이션/트랜지션**
```bash
./scripts/create_ux_request.sh "페이지 전환 애니메이션 추가" "페이지 이동 시 부드러운 애니메이션이 있으면 좋겠어요" "animation"
```

**📐 레이아웃/배치**
```bash
./scripts/create_ux_request.sh "버튼 간격 조정" "하단 버튼들이 너무 붙어있어서 실수로 누르게 돼요" "layout"
```

**🧭 내비게이션/플로우**
```bash
./scripts/create_ux_request.sh "뒤로가기 버튼 추가" "특정 화면에서 뒤로가기가 어려워요" "navigation"
```

**♿ 접근성**
```bash
./scripts/create_ux_request.sh "버튼 크기 확대" "터치 타겟이 너무 작아서 누르기 어려워요" "accessibility"
```

### 📊 자동 설정되는 내용

✅ **자동 라벨링**: 카테고리에 따른 적절한 라벨 자동 추가
✅ **우선순위**: UX 개선은 기본적으로 낮은 우선순위로 설정
✅ **템플릿**: 일관된 형식의 이슈 설명 자동 생성
✅ **추적**: 생성된 JIRA 링크 즉시 제공

### 🏷️ 사용 가능한 카테고리

| 카테고리 | 용도 | 우선순위 |
|---------|------|----------|
| `font` | 폰트 크기, 타이포그래피 | 낮음 |
| `color` | 색상, 테마, 대비 | 낮음 |
| `animation` | 애니메이션, 트랜지션 | 낮음 |
| `layout` | 레이아웃, 간격, 배치 | 낮음 |
| `navigation` | 내비게이션, 사용자 플로우 | 낮음 |
| `accessibility` | 접근성, 사용성 | **중간** |
| `general` | 기타 UX 개선 | 낮음 |

### 🎯 Claude Code에서 직접 요청하기

Claude Code와 대화할 때 다음과 같이 요청하면 **자동으로 JIRA에 추가**됩니다:

```
"메인 화면 폰트가 너무 작은 것 같아요. JIRA에 개선 요청 추가해주세요."
"다크모드 색상이 잘 안 보여요. 이것도 개선 사항으로 등록해주세요."
"페이지 전환 애니메이션 추가하고 싶어요."
```
