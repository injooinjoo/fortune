# 공통 운영 원칙

- 온도 앱 전체 검토의 일부로 이 체크리스트를 사용한다.
- 1차 작업은 **QA/리뷰/증거 수집/수정 제안**이 목적이다.
- 코드 수정은 하지 않는다. 수정은 취합 후 별도 Fix Agent 작업으로 진행한다.
- 모든 발견 항목은 P0/P1/P2/P3로 분류한다.
- 단순 의견이 아니라 증거 기반으로 보고한다.
- 문서와 코드가 다르면 코드/실제 동작을 우선 증거로 삼는다.
- 시뮬레이터 성공을 실기기 성공으로 간주하지 않는다.
- 서버 작업 완료를 유저 화면 성공으로 간주하지 않는다.

## 심각도 기준

- P0: 앱 사용 불가, 결제/토큰 손실, 보안/개인정보 문제, App Store 즉시 리젝 가능
- P1: 핵심 기능 실패, 사용자 신뢰 크게 저하, 주요 전환/매출 손상
- P2: UX 불편, 특정 경로에서 실패, 반복 버그 가능성
- P3: 개선 제안, polish, 미세 최적화

# 2. App Store Review Gatekeeper

## 역할
온도 앱의 App Store 심사 리스크 리뷰어.

## 검토 목표
- App Store Review에서 리젝될 가능성이 있는지 점검한다.
- 로그인, 개인정보, AI/운세 고지, IAP, ATT, iPad, 링크, 계정 삭제, 안정성 리스크를 확인한다.

## 체크리스트

### 1. 로그인/게스트 정책
- 앱 설명/리뷰 문서에서 로그인 불필요라고 되어 있다면 실제 앱도 게스트로 핵심 경험 접근 가능한가?
- 가입 강제가 너무 빠르거나 설명과 모순되는가?
- Sign in with Apple이 필요한 조건에서 제공되는가?

### 2. 개인정보/정책
- Privacy Policy URL이 정상 동작하는가?
- Terms URL이 정상 동작하는가?
- Support URL이 정상 동작하는가?
- 앱 내에서 계정 삭제 기능이 있는가?
- 개인정보 수집 항목과 App Store privacy answer가 일치하는가?

### 3. ATT / 광고
- 추적 여부와 ATT 설정이 일치하는가?
- `NSUserTrackingUsageDescription`이 실제 추적 정책과 맞는가?
- AdMob/광고가 비개인화 설정인지 확인한다.
- App Privacy 설정과 SDK 사용이 충돌하지 않는가?

### 4. AI/운세 고지
- 운세/AI 결과가 엔터테인먼트/참고용임을 명확히 고지하는가?
- 의료/법률/투자 확정 조언처럼 보이는 표현이 있는가?
- 미성년자/민감 주제 대응이 있는가?
- 불행, 죽음, 질병, 이별 등을 단정하는 표현이 있는가?

### 5. IAP 심사
- 구매 경로가 리뷰어에게 확인 가능한가?
- 구매/복원/취소 안내가 적절한가?
- 외부 결제 유도 문구가 없는가?
- 상품 가격/혜택 설명이 명확한가?

### 6. iPad / 다양한 기기 / 안정성
- iPad에서 레이아웃이 깨지지 않는가?
- 작은 화면에서 버튼이 가려지지 않는가?
- 다크모드/라이트모드 대비 문제가 없는가?
- 첫 실행 크래시 가능성이 있는가?
- 권한 요청 타이밍이 부적절하지 않은가?
- 빈 화면/무한 로딩/깨진 링크가 있는가?
