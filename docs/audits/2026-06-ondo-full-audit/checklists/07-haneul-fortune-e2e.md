# 공통 운영 원칙

- 온도 앱 전체 검토의 일부로 이 체크리스트를 사용한다.
- 1차 작업은 **QA/리뷰/증거 수집/수정 제안**이 목적이다.
- 코드 수정은 하지 않는다. 수정은 취합 후 별도 Fix Agent 작업으로 진행한다.
- 모든 발견 항목은 P0/P1/P2/P3로 분류한다.
- 단순 의견이 아니라 증거 기반으로 보고한다.
- 문서와 코드가 다르면 코드/실제 동작을 우선 증거로 삼는다.
- 시뮬레이터 성공을 실기기 성공으로 간주하지 않는다.
- 서버 작업 완료를 유저 화면 성공으로 간주하지 않는다.

## 심각도 기준

- P0: 앱 사용 불가, 결제/토큰 손실, 보안/개인정보 문제, App Store 즉시 리젝 가능
- P1: 핵심 기능 실패, 사용자 신뢰 크게 저하, 주요 전환/매출 손상
- P2: UX 불편, 특정 경로에서 실패, 반복 버그 가능성
- P3: 개선 제안, polish, 미세 최적화

# 7. Haneul Fortune E2E QA Reviewer

## 검토 목표
- 하늘이 운세 선택부터 설문, 생성, 취소, 결과 표시, 과거 결과 재열람까지 전체 흐름을 검토한다.
- 실제 유저가 운세를 끝까지 받을 수 있는지, 중간 실패 시 토큰/상태가 안전한지 확인한다.

## 체크리스트

### 1. 진입 경로
- 채팅에서 운세 추천 진입
- 운세 목록 bottom sheet
- fortune picker hero carousel
- 카드/아이콘/설명 구분성

### 2. 운세 선택
- 선택한 운세 타입이 정확히 전달되는가?
- 비용 표시가 명확한가?
- 무료/유료 구분이 명확한가?
- 선택 후 다음 단계가 자연스러운가?

### 3. 설문 흐름
- 질문이 너무 많지 않은가?
- 한 질문에 한 개념만 묻는가?
- 이미 프로필에 있는 정보를 다시 묻지 않는가?
- required field가 Edge Function input과 일치하는가?
- 뒤로가기/취소/수정이 가능한가?

### 4. 생성 흐름
- 로딩 상태가 명확한가?
- 장시간 생성 시 유저가 불안하지 않은가?
- 중간 취소가 가능한가?
- timeout/retry 처리가 있는가?
- 실패 시 토큰 환불이 정확한가?

### 5. 결과 표시 / 재열람
- embedded result card가 정상 표시되는가?
- fullscreen result가 정상 표시되는가?
- 차트/아이콘/섹션이 깨지지 않는가?
- 긴 결과를 읽기 편한가?
- 한국어 결과 품질이 좋은가?
- 과거 결과를 다시 열 수 있는가?
- 같은 운세 다시 생성과 캐시 보기 정책이 명확한가?
- 앱 재시작 후 결과가 유지되는가?

### 6. 특수 케이스
- 네트워크 실패
- 토큰 부족
- 생성 중 앱 종료
- 생성 중 방 나가기
- 작은 화면
- 다크모드
