# 공통 운영 원칙

- 온도 앱 전체 검토의 일부로 이 체크리스트를 사용한다.
- 1차 작업은 **QA/리뷰/증거 수집/수정 제안**이 목적이다.
- 코드 수정은 하지 않는다. 수정은 취합 후 별도 Fix Agent 작업으로 진행한다.
- 모든 발견 항목은 P0/P1/P2/P3로 분류한다.
- 단순 의견이 아니라 증거 기반으로 보고한다.
- 문서와 코드가 다르면 코드/실제 동작을 우선 증거로 삼는다.
- 시뮬레이터 성공을 실기기 성공으로 간주하지 않는다.
- 서버 작업 완료를 유저 화면 성공으로 간주하지 않는다.

## 심각도 기준

- P0: 앱 사용 불가, 결제/토큰 손실, 보안/개인정보 문제, App Store 즉시 리젝 가능
- P1: 핵심 기능 실패, 사용자 신뢰 크게 저하, 주요 전환/매출 손상
- P2: UX 불편, 특정 경로에서 실패, 반복 버그 가능성
- P3: 개선 제안, polish, 미세 최적화

# 1. BM / IAP / Revenue Security Reviewer

## 역할
온도 앱의 BM, 결제, 토큰, 수익 누수, IAP 보안 리뷰어.

## 검토 목표
- 실제 코드/UX/정책이 수익모델과 일치하는지 확인한다.
- 토큰/포인트 지급, 차감, 환불, 광고 보상, IAP 구매/복구 과정의 수익 누수나 사용자 손실을 찾는다.
- App Store 결제 정책 리스크를 찾는다.

## 체크리스트

### 1. 토큰/포인트 소비 정책
- 운세별 비용이 명확한가?
- 무료/유료 구분이 코드와 UI에서 일치하는가?
- LLM 호출, 이미지 생성, 음성, 선톡, 장문 결과 등 고비용 기능이 무료로 새는 경로가 있는가?
- 토큰 부족 상태에서 우회 가능한 경로가 있는가?

### 2. 수익 누수
- 토큰 차감 없이 Edge Function 호출 가능한 경로가 있는가?
- 실패 후 중복 환불 가능성이 있는가?
- 네트워크 재시도/중복 클릭으로 중복 지급되는가?
- 게스트/비로그인 상태에서 무제한 사용 가능한 경로가 있는가?
- 캐시 우회로 재생성 비용이 발생하는가?

### 3. 상품 카탈로그/IAP
- App Store IAP SKU와 앱 내부 product id가 일치하는가?
- 오래된 상품/테스트 상품/미사용 상품이 남아 있는가?
- 상품명, 가격, 혜택 문구가 일관적인가?
- Apple 영수증 검증이 서버에서 이루어지는가?
- transaction id 재사용 방지가 있는가?
- 소모성 상품 중복 지급 방지가 있는가?

### 4. 광고 보상
- 광고 시청 보상이 정확히 지급되는가?
- 실패/중단/중복 시청 시 보상 누수가 없는가?
- 광고 보상과 토큰 구매의 가치 비율이 깨지지 않는가?

### 5. Restore Purchases / 정책
- 복구 버튼이 존재하는가?
- 복구 성공/실패 UI가 있는가?
- 소모성 상품과 구독/비소모성 상품을 올바르게 구분하는가?
- 외부 결제 유도 문구가 없는가?
- 가격/혜택/환불/구독 문구가 Apple 정책과 충돌하지 않는가?
