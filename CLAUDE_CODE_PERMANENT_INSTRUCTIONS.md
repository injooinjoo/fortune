# 🤖 Claude Code 영구 자동화 지침

## 🎯 **필수 작업 플로우** (절대 건너뛰지 말 것)

### 1️⃣ **모든 불만/개선 요청 자동 JIRA 등록**
사용자가 다음과 같은 내용을 말하면 **즉시 자동으로** `./scripts/parse_ux_request.sh` 실행:

✅ **자동 감지 키워드**:
- 불만: "~이 문제야", "~이 이상해", "버그같아", "안돼", "작동안해"
- 개선: "~하면 좋겠어", "~으로 바꿔줘", "개선", "수정"
- UX: "사용하기 어려워", "불편해", "보기 힘들어", "터치하기 어려워"
- 디자인: "폰트", "색상", "크기", "간격", "레이아웃", "애니메이션"

### 2️⃣ **문제 해결 후 자동 JIRA 완료 처리**
코드 수정 완료 시 **반드시** `./scripts/git_jira_commit.sh` 실행:

```bash
./scripts/git_jira_commit.sh "해결 내용 설명" "JIRA이슈번호" "done"
```

## 🔄 **완전 자동화 워크플로우**

### Step 1: 사용자 불만 감지
```
사용자: "홈 화면 로딩이 너무 오래 걸려"
Claude: [즉시 parse_ux_request.sh 실행]
Claude: "KAN-XX 이슈 생성 완료! 문제를 해결해보겠습니다."
```

### Step 2: 문제 분석 및 해결
```
Claude: [코드 분석 및 수정 진행]
Claude: [문제 해결 후]
```

### Step 3: 자동 완료 처리
```
Claude: [git_jira_commit.sh 실행으로 Git 커밋 + JIRA Done 처리]
Claude: "해결 완료! JIRA에서도 완료 처리했습니다."
```

## 📋 **Claude Code 필수 실행 규칙**

### ✅ **항상 해야 할 것**
1. 불만/개선 요청 감지 시 → **즉시** JIRA 등록
2. 문제 해결 완료 시 → **반드시** JIRA 완료 처리
3. 모든 단계에서 사용자에게 JIRA 링크 제공
4. 해결 과정을 JIRA 코멘트에 자동 기록

### ❌ **절대 하지 말 것**
1. 불만을 들고도 JIRA 등록 안 하기
2. 문제 해결하고도 JIRA 완료 처리 안 하기
3. 수동으로 처리하라고 사용자에게 떠넘기기

## 🎯 **실제 사용 예시**
